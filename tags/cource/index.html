Scala Evaluation strategy and Termination Sep 21 2012

Following Functional Programming Principles in Scala Course Lesson 1

I would like to demonstrate Scala evaluation models
and share 2 examples of code in Scala.

First is using default call-by-value evaluation.
Which means arguments will be calculated before the function call
This can create a problem for the function which is indefinite - like loop

In the following example second parameter is using call-by-name evaluation
This mode can be secified by adding =>
Which means result of it will not be calculated unless it is used inside the function
it works even with infinit funtions
